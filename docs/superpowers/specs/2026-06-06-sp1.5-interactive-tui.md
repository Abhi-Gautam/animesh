## 1. Context

SP-1 shipped the durable substrate (SQLite + refinery + FTS5 + AniList client). v0.3 is a competent CLI. The design handoff makes the project's primary UX an interactive ratatui-based TUI inspired by lazygit / htop / k9s — three panes (Today / Late / Backlog), live detail pane, command palette, full keymap, optional cover art.

SP-1.5 builds the TUI on top of SP-1's substrate. The CLI subcommands stay unchanged for scripts and headless use.

## 2. Locked decisions

### Substrate (preserved from SP-1)
- **Durable store:** SQLite via rusqlite + refinery migrations (no change).
- **User-edited preferences** (timezone, accent, notify mode, quiet hours): `$XDG_CONFIG_HOME/animesh/config.toml`. Not shipped in v0.4 — env vars cover everything for now.
- **Portable export** (`animesh export`): future; the "user owns their data" ethos is satisfied today by SQLite being a plain local file.

### Fuzzy
- FTS5 narrows candidate set (sub-ms over any library).
- `nucleo` re-ranks the top N for typo tolerance and the interactive feel.

### Bucketing algorithm (canonical)

```
let seen = watch_progress.seen.unwrap_or(0);
let next_ep = cache.next_episode_number;
let total = cache.total_episodes;
let airs_at = cache.next_episode_airs_at;

const TODAY_WINDOW_SECS = env("ANIMESH_TODAY_WINDOW_HOURS", 24) * 3600;
const LATE_WINDOW_SECS  = env("ANIMESH_LATE_WINDOW_HOURS",  48) * 3600;

// Fully watched → hidden from all panes.
if let Some(t) = total { if seen >= t { return None; } }

// Behind by N → Backlog with that count.
let behind = next_ep.map(|n| (n - 1).saturating_sub(seen)).unwrap_or(0);
if behind > 0 { return Some(Backlog { behind }); }

// Caught up — pane depends on the next episode's airing time.
if let Some(at) = airs_at {
    if at > now && at <= now + TODAY_WINDOW_SECS { return Some(Today); }
    if at < now && at >= now - LATE_WINDOW_SECS && next_ep.map_or(false, |n| seen < n) {
        return Some(Late);
    }
}

// Not fully done but no near-term air time → "Backlog" with behind=0 (e.g. dandadan finale waiting).
if total.map_or(false, |t| seen < t) { return Some(Backlog { behind: 0 }); }

// Caught up, nothing imminent, unknown total → hidden.
None
```

### Schema delta — V0002

- New table **`watch_progress`**: `(source, source_id, seen, updated_at)`, PK `(source, source_id)`. Source-neutral so SP-7 generalization works.
- Extend **`metadata_cache`** with: `cover_image_url`, `description`, `score`, `studios`, `streaming_links_json`. All ephemeral; revoked alongside the rest of the cache.

### Launch & lifecycle

- `animesh` with no subcommand → launches the TUI. All existing subcommands preserved.
- TUI opens alternate screen, raw mode, hides cursor; panic-hook restores terminal cleanly.
- One owned `Db` for the runloop; one owned `AniListClient`.
- On launch: kick off a background refresh of stale follows. UI renders immediately from local cache; rows update as refreshes return.
- 30-second tick re-renders relative times (the "in 3h" / "2h ago" labels) and the wall-clock.

### Module structure

```
src/tui/
  mod.rs        // public Tui::run() entry; panic restore guard
  app.rs        // App state machine, event loop, action dispatch
  model.rs      // Show, Pane, derived view objects
  pane.rs       // bucketing algorithm + tests
  view.rs       // ratatui frame composition
  detail.rs     // detail pane rendering
  palette.rs    // command palette (nucleo over FTS5-scoped candidates)
  toast.rs      // toast queue
  help.rs       // help overlay
```

### Cover art

Half-block (`▀`) renderer using `image` crate is the v0.5 target. v0.4 ships a text placeholder (color-fill block of the show's pal accent) so the layout is right and the upgrade is purely additive.

### Keymap (final)

| Key(s) | Action |
|---|---|
| `j` / `k` / `↓` / `↑` | Move within focused panel (wraps) |
| `Tab` / `Shift-Tab` | Cycle focused panel |
| `1` / `2` / `3` | Jump to Today / Late / Backlog |
| `h` / `l` / `←` / `→` | Switch focused panel |
| `Enter` | Confirm in overlays (no detail-screen change — detail is always live) |
| `w` | Increment seen; toast |
| `s` | Snooze stub → toast (full snooze table in SP-3) |
| `d` | Drop selected → toast |
| `g` | Open primary streaming URL in browser (`open` crate) |
| `a` / `:` / `/` | Open command palette |
| `?` | Help overlay |
| `Esc` | Close overlay |
| `q` / `Ctrl-C` | Quit (restore terminal) |

### Deps added in v0.4

| Crate | Use |
|---|---|
| `ratatui` | TUI layout / widgets |
| `nucleo` | Palette fuzzy re-ranking |
| `open` | `g` key — launch browser |

Deferred: `ratatui-image` (v0.5 cover art), `notify-rust` (SP-3), `toml` (when config file lands).

## 4. Tests

- `pane.rs` carries property tests over `(seen, total, next_ep, airs_at, now)` → expected `Pane`. The bucketing rule must be deterministic and well-defined for every combination.
- `palette.rs` carries unit tests for the fuzzy ranker.
- TUI rendering tests use ratatui's `TestBackend` to assert on the rendered buffer.
- Smoke test: build the binary, run `animesh` non-interactively (we send `q` immediately via a piped backend) to confirm the alternate-screen toggle, render, and quit path don't panic.
